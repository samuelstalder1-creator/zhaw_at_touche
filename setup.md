# Setup Reference

This file explains the experiment matrix: what each family is testing, how the setups relate to each other, and what concepts they use. Metrics and confusion matrices live in `results.md`.

---

## The Core Question

Every setup is answering one of two questions:

1. **Can a classifier detect ad language directly from query + response?**
2. **Does comparing the response to a neutral rewrite add signal?**

The neutral rewrite (generated by Gemini or Qwen) is a version of the response with any promotional language removed. If the original response contains an ad, the delta between it and the neutral rewrite should expose that ad.

---

## Experiment Families

### Family 1 — Fine-tuned classifiers (setup4, setup6–setup12)

A transformer encoder sees the input and outputs a binary label. The backbone is fine-tuned end-to-end.

**Input format variants:**

| Format | Setups | What the model sees |
| --- | --- | --- |
| `query_response` | setup6, setup6-qwen, setup8, setup9, setup10, setup11, setup12 | query + response only |
| `query_neutral_response` | setup7, setup7-qwen | query + neutral rewrite + response |
| `query_reference_rag_response` | setup4 | query + labeled neutral reference + response |

The `query_response` family answers: **how far can a classifier get without the neutral rewrite?** Setup6 (RoBERTa) is the baseline. Setup12 (DistilRoBERTa) is smaller and faster. Setup10 (ALBERT) is the most compact. Setup8/9 test DeBERTa-v3, which is theoretically stronger but prone to training instability in this repo.

The `query_neutral_response` family answers: **does injecting the neutral into the prompt help?** Setup7 uses Longformer to handle the longer combined input. Setup7-qwen swaps the Gemini neutral for a Qwen-generated one.

**Provider variants (`-qwen` suffix):** these use data from `data/generated/qwen/` instead of Gemini. For `query_response` setups the neutral is not in the prompt, so the provider affects only training data alignment. For `query_neutral_response` setups the provider changes what the model reads.

---

### Family 2 — Embedding divergence (setup100–102)

No fine-tuning. A frozen sentence encoder embeds the response and neutral separately, then a threshold on the cosine distance decides the label.

| Setup | Aggregation | Threshold target |
| --- | --- | --- |
| setup100 | sentence mean | macro F1 |
| setup101 | top-3 sentence mean | positive F1 (higher recall) |
| setup102 | top-3 sentence mean, larger encoder (BGE-large) | positive F1 |

These are training-free baselines. They fail because cosine distance is a single scalar that discards directional information — two responses can drift the same angle from their neutral while one is promotional and the other is not.

---

### Family 3 — Learned embedding features (setup103–104, setup113–114)

Instead of thresholding a distance, a logistic regression is trained on the embedding vectors themselves. The encoder stays frozen; only the logistic regression weights are learned. This is the key fix over family 2: the classifier learns *which directions in embedding space correspond to advertising*.

| Setup | Features | Dim |
| --- | --- | --- |
| setup103 | `response_emb − gemini_emb` | 768 |
| setup104 | `[response_emb \| gemini_emb \| delta_gemini]` | 2304 |
| setup113 | `[delta_gemini \| delta_qwen]` | 1536 |
| setup114 | `[response_emb \| gemini_emb \| qwen_emb \| delta_gemini \| delta_qwen]` | 3840 |

Setup103 is the single-provider residual. Setup104 adds absolute embedding positions on top. Setup113 and setup114 extend both ideas to dual-provider neutrals (Gemini + Qwen), testing whether two independent neutral sources add signal.

Setup110 is a failed variant that collapsed the same embedding information into 6 cosine scalars — the scalar bottleneck discards most of what makes setup103 work.
Setup111 keeps the same six cosine scalars but removes classifier training entirely: it scores each row as mean(response-to-anchor distances) minus mean(anchor-to-anchor distances), then fits only a threshold.

---

### Family 4 — Cross-encoder (setup105, setup105_1)

The response and neutral are concatenated into one sequence so every transformer layer can attend across both texts simultaneously. This is the most expressive neutral-aware design: instead of comparing fixed vectors after encoding, the model can attend to the exact tokens that differ.

| Setup | Backbone | Issue |
| --- | --- | --- |
| setup105 | DeBERTa-v3-base | collapsed to all-positive; also missing eval config |
| setup105_1 | RoBERTa-base | retry with stable backbone + class weight compensation |

The cross-encoder is theoretically superior to family 3 for localized ad insertions because it can identify the specific tokens that diverge between response and neutral rather than comparing compressed fixed vectors.

---

### Archived — Sentence-delta classifier (setup106)

Operates at sentence level rather than document level. Sentences in the response are aligned to sentences in the neutral via Hungarian matching. Per-pair features (`[sent_emb | neutral_sent_emb | delta]`) are passed through a classifier, and sentence-level probabilities are aggregated into one document score. This preserves localized evidence that document-level approaches average away.

## Optimization Concepts

| Concept | Where used | Purpose |
| --- | --- | --- |
| `lr_scheduler: linear` | setup10–12 | decays LR linearly after warmup, stable default |
| `lr_scheduler: cosine_with_warmup` | setup8, setup9, setup105 | smoother decay, standard for DeBERTa |
| `warmup_ratio` | most setups | gradual LR ramp to avoid early instability |
| `max_grad_norm` | DeBERTa setups | gradient clipping, key anti-collapse control |
| `optimizer_eps: 1e-6` | DeBERTa setups | raised from default `1e-8` for numerical stability |
| `layerwise_lr_decay` | setup9 | lower LR in early layers, conservative fine-tuning |
| `freeze_embeddings_epochs` | setup9 | keeps embedding layer frozen at start |
| `positive_class_weight_scale` | setup7, setup105_1 | compensates for 70/30 class imbalance |
| `gradient_checkpointing` | setup4 | trades compute for lower memory |
| `pad_to_max_length` | setup7, setup7-qwen | consistent padding for long-context batches |
