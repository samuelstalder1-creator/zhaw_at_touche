# Setup Reference

This file explains the experiment matrix: what each family is testing, how the setups relate to each other, and what concepts they use. Metrics and confusion matrices live in `results.md`.

---

## The Core Question

Every setup is answering one of two questions:

1. **Can a classifier detect ad language directly from query + response?**
2. **Does comparing the response to a neutral rewrite add signal?**

The neutral rewrite (generated by Gemini or Qwen) is a version of the response with any promotional language removed. If the original response contains an ad, the delta between it and the neutral rewrite should expose that ad.

---

## Experiment Families

### Family 1 — Fine-tuned classifiers (setup4, setup6–setup12, setup7_2-qwen, setup115–116)

A transformer encoder sees the input and outputs a binary label. The backbone is fine-tuned end-to-end.

**Input format variants:**

| Format | Setups | What the model sees |
| --- | --- | --- |
| `response_only` | setup115 | response only |
| `query_response` | setup6, setup6-qwen, setup8, setup9, setup10, setup11, setup12 | query + response only |
| `query_neutral_response` | setup7, setup7-qwen, setup7_2-qwen | query + neutral rewrite + response |
| `query_dual_neutral_response` | setup116 | query + Gemini neutral + Qwen neutral + response |
| `query_reference_rag_response` | setup4 | query + labeled neutral reference + response |

The `response_only` family isolates the value of the query. Setup115 keeps the
same RoBERTa backbone as setup6 but removes the query entirely.

The `query_response` family answers: **how far can a classifier get without the neutral rewrite?** Setup6 (RoBERTa) is the baseline. Setup12 (DistilRoBERTa) is smaller and faster. Setup10 (ALBERT) is the most compact. Setup8/9 test DeBERTa-v3, which is theoretically stronger but prone to training instability in this repo.

The `query_neutral_response` family answers: **does injecting the neutral into the prompt help?** Setup7 uses Longformer to handle the longer combined input. Setup7-qwen swaps the Gemini neutral for a Qwen-generated one. Setup7_2-qwen keeps that Qwen-backed prompt format and backbone but extends the optimization schedule and regularization to match the setup10-style classifier defaults more closely.

The `query_dual_neutral_response` family asks whether **two** independently
generated neutrals help more than one. Setup116 uses Longformer again because
both neutrals must fit into the prompt at once.

**Provider variants (`-qwen` suffix):** these use data from `data/generated/qwen/` instead of Gemini. For `query_response` setups the neutral is not in the prompt, so the provider affects only training data alignment. For `query_neutral_response` setups the provider changes what the model reads.

### Family 2 — Embedding divergence (setup100–102)

No fine-tuning. A frozen sentence encoder embeds the response and neutral separately, then a threshold on the cosine distance decides the label.

| Setup | Aggregation | Threshold target |
| --- | --- | --- |
| setup100 | sentence mean | macro F1 |
| setup101 | top-3 sentence mean | positive F1 (higher recall) |
| setup102 | top-3 sentence mean, larger encoder (BGE-large) | positive F1 |

These are training-free baselines. They fail because cosine distance is a single scalar that discards directional information — two responses can drift the same angle from their neutral while one is promotional and the other is not.

---

### Family 3 — Learned embedding features (setup103, setup103-qwen, setup103-gemma, setup104-base, setup104, setup104-qwen, setup113–114, setup117–121)

Instead of thresholding a distance, a logistic regression is trained on the embedding vectors themselves. The encoder stays frozen; only the logistic regression weights are learned. This is the key fix over family 2: the classifier learns *which directions in embedding space correspond to advertising*.

| Setup | Features | Dim |
| --- | --- | --- |
| setup103 | `response_emb − gemini_emb` | 768 |
| setup103-qwen | `response_emb − qwen_emb` | 768 |
| setup103-gemma | `response_emb − gemma4_e4b_emb` | 768 |
| setup104-base | `response_emb` | 768 |
| setup104 | `[response_emb \| gemini_emb \| delta_gemini]` | 2304 |
| setup104-qwen | `[response_emb \| qwen_emb \| delta_qwen]` | 2304 |
| setup113 | `[delta_gemini \| delta_qwen]` | 1536 |
| setup114 | `[response_emb \| gemini_emb \| qwen_emb \| delta_gemini \| delta_qwen]` | 3840 |
| setup117 | `[query_emb \| delta_gemini]` | 1536 |
| setup118 | `[query_emb \| delta_gemini \| delta_qwen]` | 2304 |
| setup119 | `response_emb − qwen_emb` | 768 |
| setup120-qwen | `centered delta_qwen + abs(delta) + cosine + ||delta||` | 1538 |
| setup120-gemma | `centered delta_gemma + abs(delta) + cosine + ||delta||` | 1538 |
| setup121-qwen | `[response_emb \| qwen_emb \| centered delta_qwen \| abs(delta) \| cosine \| ||delta||]` | 3074 |
| setup121-gemma | `[response_emb \| gemma4_e4b_emb \| centered delta_gemma \| abs(delta) \| cosine \| ||delta||]` | 3074 |

Setup103 is the single-provider Gemini residual. Setup103-qwen and
setup103-gemma apply the same residual-only design to Qwen and Gemma neutral
files. Setup104-base is the response-only control in the same frozen-encoder +
logistic-regression family, aligned to the Qwen-backed files so it can be
compared directly against setup104-qwen to measure the incremental value of
adding the neutral and residual blocks. Setup119 remains the earlier
Qwen-only residual naming used in the committed results. Setup104 and
setup104-qwen add absolute embedding positions on top for Gemini and Qwen
respectively. Setup113 and setup114 extend both ideas to dual-provider
neutrals (Gemini + Qwen), testing whether two independent neutral sources add
signal. Setup117 and setup118 inject the query embedding into the same delta
family to test whether topic context sharpens the residual signal.
Setup120-qwen and setup120-gemma keep the residual-only design but recenter
the provider-specific delta by the negative-class mean and append cheap
scalar/vector diagnostics (`abs(delta)`, cosine similarity, delta norm), while
letting validation choose the LR regularization and class-weight setting.
Setup121-qwen and setup121-gemma apply the same improvements to the full
`[response | neutral | delta]` stack.

### Family 4 — Cross-encoder (setup105, setup105_1)

The response and neutral are concatenated into one sequence so every transformer layer can attend across both texts simultaneously. This is the most expressive neutral-aware design: instead of comparing fixed vectors after encoding, the model can attend to the exact tokens that differ.

| Setup | Backbone | Issue |
| --- | --- | --- |
| setup105 | DeBERTa-v3-base | collapsed to all-positive; also missing eval config |
| setup105_1 | RoBERTa-base | retry with stable backbone + class weight compensation |

The cross-encoder is theoretically superior to family 3 for localized ad insertions because it can identify the specific tokens that diverge between response and neutral rather than comparing compressed fixed vectors.

### Family 5 — Scalar anchor baselines (setup110–111)

These setups still use Gemini + Qwen at the same time, but they collapse the
comparison down to six cosine distances between `query`, `response`, Gemini,
and Qwen. Setup110 learns a logistic regression over those six scalars.
Setup111 uses the handcrafted score `response_drift - anchor_cohesion` and
calibrates only a threshold. They are useful controls because they test whether
dual-provider information helps once most of the vector geometry has been
discarded.

### Descriptor Only — Sentence-delta classifier (setup106)

Operates at sentence level rather than document level. Sentences in the response are aligned to sentences in the neutral via Hungarian matching. Per-pair features (`[sent_emb | neutral_sent_emb | delta]`) are passed through a classifier, and sentence-level probabilities are aggregated into one document score. This preserves localized evidence that document-level approaches average away.

## Optimization Concepts

| Concept | Where used | Purpose |
| --- | --- | --- |
| `lr_scheduler: linear` | setup7_2-qwen, setup10–12 | decays LR linearly after warmup, stable default |
| `lr_scheduler: cosine_with_warmup` | setup8, setup9, setup105 | smoother decay, standard for DeBERTa |
| `warmup_ratio` | most setups | gradual LR ramp to avoid early instability |
| `max_grad_norm` | DeBERTa setups | gradient clipping, key anti-collapse control |
| `optimizer_eps: 1e-6` | DeBERTa setups | raised from default `1e-8` for numerical stability |
| `layerwise_lr_decay` | setup9 | lower LR in early layers, conservative fine-tuning |
| `freeze_embeddings_epochs` | setup9 | keeps embedding layer frozen at start |
| `positive_class_weight_scale` | setup7, setup7-qwen, setup7_2-qwen, setup105_1, setup116 | compensates for 70/30 class imbalance |
| `gradient_checkpointing` | setup4 | trades compute for lower memory |
| `pad_to_max_length` | setup7, setup7-qwen, setup7_2-qwen, setup116 | consistent padding for long-context batches |
| `delta_centering: negative_mean` | setup120-qwen, setup120-gemma, setup121-qwen, setup121-gemma | subtracts the mean negative-class provider delta before classification |
| `append_delta_abs`, `append_pairwise_cosine`, `append_delta_norm` | setup120-qwen, setup120-gemma, setup121-qwen, setup121-gemma | adds provider-drift diagnostics without discarding the full residual vector |
| `lr_c_values`, `lr_class_weight_options` | setup120-qwen, setup120-gemma, setup121-qwen, setup121-gemma | lets validation select the LR regularization and whether class balancing is still helpful |
