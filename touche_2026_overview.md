# Touché @ CLEF 2026 Overview

Touché @ CLEF 2026 contains four shared tasks:

1. Fallacy Detection
2. Causality Extraction
3. Generalizability of Argument Identification in Context
4. Advertisement in Retrieval-Augmented Generation

This repository is aligned with the fourth task, **Advertisement in Retrieval-Augmented Generation**, which appears in its second edition in 2026. The task studies how advertisements can be detected and removed from retrieval-augmented generation (RAG) responses.

## Advertisement in Retrieval-Augmented Generation

The shared task is split into three subtasks:

1. **Ad classification**
   Input: `response`, `query`, and auxiliary metadata
   Output: binary label `contains advertisement` (`0/1`)
   This is a response-level classification task.
2. **Span detection**
   Input: a response containing advertising content
   Output: character offsets for the ad spans
   This is a span detection task.
3. **Ad removal / rewriting**
   Input: the original response and the detected offsets
   Output: a rewritten response without advertisements that remains fluent and relevant

## Dataset

The task is based on the **Webis Generated Native Ads 2025** dataset.

Main properties:

- official `train`, `dev`, and `test` splits
- responses generated by RAG search engines
- synthetic advertisement insertions
- query metadata such as `query`, `meta_topic`, and `search_engine`
- character-span annotations for inserted ads

The responses originate from real RAG-style systems:

- Brave
- Copilot
- Perplexity
- YouChat

This makes the task realistic: the responses are genuine RAG outputs, while the advertising content is injected in a controlled and annotated way.

## Data Generation Pipeline

The task description and accompanying paper describe a pipeline with these steps:

1. Collect user queries
2. Generate responses with RAG search engines
3. Insert advertisements synthetically
4. Annotate the inserted spans
5. Create the final train/dev/test splits

The 2026 dataset contains:

- 30,731 responses
- 9,062 queries
- multiple topics
- both ad and non-ad examples

## Dataset Structure

At response level, the data includes fields such as:

- `response`
- `query`
- `meta_topic`
- `search_engine`
- `label`

At span level, the data includes fields such as:

- `spans`
- `sentence_spans`
- `advertiser`
- `item`

The task description also motivates sentence-level views of the problem, for example:

- sentence pairs
- boundary detection
- transition detection

## Why This Task Matters

The task is motivated by the fact that modern language systems can produce advertising content that is:

- contextually integrated
- difficult to detect
- not transparently disclosed

In RAG settings this can appear as:

- native ads
- product placement
- persuasive inserts
- contextual advertising

The 2026 task therefore evaluates detection, localization, and rewriting of advertising content in generated answers.

## Injection Types in the 2026 Setting

The task uses multiple prompt-injection styles with different stealth levels:

- `base`
- `end`
- `rewrite`
- `question`
- `call_to_action`

These simulate different forms of promotional content, ranging from subtle contextual integration to more explicit persuasive prompts.

## Scientific Goal

From a research perspective, the task asks how well systems can:

- detect whether a generated answer contains advertising
- localize the ad spans precisely
- rewrite the answer so that it remains useful without the promotional content

This makes the task relevant for trustworthy RAG systems, transparent retrieval interfaces, and the evaluation of persuasion in generated content.
